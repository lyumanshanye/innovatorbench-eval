InnovatorBench 数据结构

661 条轨迹，每条轨迹由 nodes/ 目录下的 JSON 文件组成。节点类型：root（任务描述）、react（工具调用，含 action + observation + response）、planning（子目标规划）、judge（rubric 评估）、summary（上下文压缩）。每个 react 节点包含：action.action_type（28种：run_command, edit_file, open_file, eval, create_session, sleep, think, plan 等）、response.usage（token统计）、response.reasoning（部分模型有）、observation（执行结果）。eval 节点含 overall_score(0-100) 和 already_eval_num(最多3次)。

VI区分度检验算法

计算出指标数值后，需要验证每条指标是否真的能区分"好 Agent"和"差 Agent"。以下是从基础到高阶的区分度检验方法体系。

Level 1：单指标区分度（必做）

1.1 Kruskal-Wallis H 检验

目的：检验该指标在不同模型间是否有显著差异（非参数 one-way ANOVA）
方法：scipy.stats.kruskal(*[metric_values_per_model])
判据：p < 0.05 → 指标有区分度；H 值越大区分度越强
适用：所有 96 条指标的第一轮筛选

1.2 Effect Size — η² (Eta-squared)

目的：量化区分效果的大小（不只看是否显著，还看效果多大）
方法：η² = H / (N - 1)，其中 H 是 Kruskal-Wallis 统计量，N 是样本总数
判据：η² < 0.01 弱，0.01–0.06 中，> 0.06 强
用途：在显著指标中排序优先级

1.3 Spearman 秩相关（指标 vs 最终得分）

目的：检验指标值与任务最终得分的单调相关性
方法：scipy.stats.spearmanr(metric_values, overall_scores)
判据：|ρ| > 0.3 且 p < 0.05 → 指标与成功相关
用途：验证指标是否"方向正确"（好指标值→高得分）

Level 2：指标间关系（推荐）

2.1 指标冗余度 — 指标间 Spearman 相关矩阵

目的：发现高度相关的指标对（|ρ|>0.8），可能测量同一维度，可合并
方法：计算所有可算指标对的 Spearman 矩阵，绘制 clustermap
判据：|ρ| > 0.8 的指标对需审查是否冗余
工具：seaborn.clustermap + scipy hierarchical clustering

2.2 PCA / Factor Analysis

目的：将 26 个可算指标降维，看实际独立的维度有几个
方法：sklearn.decomposition.PCA + 观察 explained_variance_ratio
判据：如果前 4–6 个主成分解释 >80% 方差 → 指标体系有较好的维度覆盖
输出：每个主成分的 loading 向量 → 解释"这个维度代表什么能力"

Level 3：预测能力（高阶）

3.1 Random Forest Feature Importance

目的：用全部指标预测 overall_score，看哪些指标贡献最大
方法：RandomForestRegressor(n_estimators=500).fit(X_metrics, y_score)
输出：feature_importances_ 排序 → Top-K 最有预测力的指标
优势：自动处理非线性关系和指标间交互

3.2 Leave-One-Out 指标消融

目的：移除某指标后预测能力下降多少 → 该指标的独立贡献
方法：对每个指标 i，计算 R²_full - R²_{without_i}
判据：ΔR² > 0.02 → 该指标有独立贡献；ΔR² ≈ 0 → 可被其他指标替代

Level 4：模型诊断能力（最终验证）

4.1 Friedman's Test + Nemenyi Post-hoc

目的：在每条指标上做跨模型排名检验，确认指标是否产生与 overall_score 一致的模型排序
方法：scipy.stats.friedmanchisquare → Nemenyi critical difference diagram
输出：CD (Critical Difference) 图 → 哪些模型在该指标上显著不同

4.2 指标体系的诊断一致性 (Coherence Score)

目的：验证整个指标体系是否能产生一致且有信息量的模型画像
方法：对每个模型，用全部指标生成"能力剖面向量"，计算模型间的距离矩阵，与 overall_score 排名的 Kendall τ 做对比
判据：τ > 0.7 → 指标体系与最终成绩高度一致，同时提供更细粒度的诊断信息
