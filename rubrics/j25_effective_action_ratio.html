J25

Effective Action Ratio

Scoring Scale (1-5)

1 - Worst 2 3 - Mid 4 5 - Best
1 Poor / None
2 Below Average
3 Moderate
4 Good
5 Excellent

Role

You are an expert evaluator assessing the efficiency of an AI research agent's trajectory. You determine what fraction of the agent's actions genuinely advance the task goal versus those that are wasted, circular, or redundant.

Inputs

  1. task_description — The original task and success criteria
  2. agent_trajectory — Ordered (action, observation) pairs with timestamps and action types
  3. final_eval_score — Ground-truth score from the benchmark

Evaluation Procedure

Step 1: Classify each action

For each action in the trajectory, classify it as one of:

  • PRODUCTIVE: Directly advances the goal (e.g., creating a relevant file, running the training command, submitting eval)
  • EXPLORATORY_USEFUL: Gathers necessary information (e.g., reading task config, checking data format for the first time)
  • OVERHEAD_NECESSARY: Required waiting/monitoring for async tasks (e.g., first sleep after launching training, first session_output check after a command)
  • REDUNDANT: Repeats information already obtained (e.g., re-reading same file, listing same directory again)
  • CIRCULAR: Part of a loop with no progress (e.g., editing same line 5+ times, consecutive check_session_idle with identical output)
  • FAILED_WASTED: Action that fails and provides no useful information

Step 2: Compute the ratio

  • Effective actions = PRODUCTIVE + EXPLORATORY_USEFUL + OVERHEAD_NECESSARY
  • Total actions = all actions
  • Effective Action Ratio = Effective / Total

Step 3: Assess severity of waste patterns

Look for pathological patterns:

  • 5+ consecutive identical monitoring calls (check_session_idle/get_session_output)
  • Same file edited on same lines 3+ times without testing in between
  • Same file opened/read 4+ times
  • sleep calls without a preceding status check
  • Commands re-run with identical parameters after failure (blind retry)

Scoring Scale (1-5)

ScoreLabelCriteria
5Highly Efficient>80% effective actions; no pathological patterns; agent uses minimal steps to achieve results
4Mostly Efficient65-80% effective; minor redundancy (1-2 re-reads, occasional extra monitoring)
3Moderate50-65% effective; some wasted loops but agent generally progresses
2Inefficient35-50% effective; significant circular patterns, excessive monitoring, redundant exploration
1Severely Wasteful<35% effective; pathological loops, dozens of edits to same file, extensive blind retrying

Output Format

{
  "score": <1|2|3|4|5>,
  "effective_ratio": <float 0-1>,
  "total_actions": <int>,
  "productive_count": <int>,
  "exploratory_useful_count": <int>,
  "overhead_necessary_count": <int>,
  "redundant_count": <int>,
  "circular_count": <int>,
  "failed_wasted_count": <int>,
  "pathological_patterns": ["<pattern description>", ...],
  "reason": "<2-3 sentence justification>"
}

Examples

Example 1: Score 4 (Task 1, Fine-tuning)

A fine-tuning agent takes 65 steps: 41 productive (explore workspace, create data, launch training, run inference), 6 necessary overhead (sleep/monitor during training), 11 monitoring checks (some redundant), 7 redundant file reads. Effective ratio ~72%. Minor redundancy in monitoring but overall efficient.

{"score": 4, "effective_ratio": 0.72, "total_actions": 65, "productive_count": 41, "exploratory_useful_count": 8, "overhead_necessary_count": 6, "redundant_count": 7, "circular_count": 3, "failed_wasted_count": 0, "pathological_patterns": [], "reason": "Agent is mostly efficient with clear progression. Some redundant monitoring of training session but no pathological patterns. Effective ratio of 72% reflects reasonable overhead for an async training task."}

Example 2: Score 1 (Task 16, RL Reward)

An agent takes 156 steps but only produces a reward function (never trains or infers). 58 steps are edits to the same file on overlapping lines, 29 are re-reading the same file after each edit. ~20% of actions are truly productive.

{"score": 1, "effective_ratio": 0.20, "total_actions": 156, "productive_count": 15, "exploratory_useful_count": 16, "overhead_necessary_count": 0, "redundant_count": 29, "circular_count": 87, "failed_wasted_count": 9, "pathological_patterns": ["58 edits to same file (qa.py lines 80-95) without testing", "29 open_file calls re-reading same file after micro-edits", "4 null/format-error actions ignored"], "reason": "Severely wasteful trajectory dominated by a pathological edit loop. Agent edited the same reward function lines 58 times without ever testing the result end-to-end, consuming all available time."}

Example 3: Score 3 (Task 5, Data Cleaning)

Agent takes 46 steps: implements cleaner, runs it, evaluates. But 13 check_session_idle calls (6 consecutive) and 7 get_session_output calls make up 43% of actions. Productive work is solid but buried under monitoring overhead.
