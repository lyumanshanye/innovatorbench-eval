Same family as M78 but controls for tool_chain_depth (a step-budget proxy) plus task fixed effects. Reveals model-quality differences that survive equalizing trajectory length.

At a Glance

FieldValue
Metric IDM79
EnglishStep-Controlled Success
中文步数控制成功率
MethodAuto / Statistical (GLM)
Value rangemodel-coefficient std (≥0)
DirectionHigh std = strong residual model differentiation; near 0 = step budget explains the gap.

Formula

score ~ C(model) + tool_chain_depth + C(task_id)
M79_diff = std(β_model)

Required Inputs

overall_score, tool_chain_depth, task_id, model.

Procedure

  1. Fit OLS: score ~ C(model) + tool_chain_depth + C(task_id) on all trajectories.
  2. Take std of β_model coefficients as the M79 differentiation score.
  3. Report β_tool_chain_depth as the size of the step-budget channel.

Worked Example

Fit on 655 trajectories: R²=0.374, F=6.6. M79_diff = 9.236; β_tool_chain_depth = +0.011. Top-3: kimi-k2.5(+20.15), glm47(+8.95), glm47_data_w_codeact(+3.35). Bottom-3: grok-4(−13.26), grok-code-fast-1(−13.x), …

Edge Cases

  • tool_chain_depth is a noisy proxy for step count — replace with raw step_count for benchmarks where the difference matters.
  • Same multicollinearity warning as M78.

Output Keys

  • M79_diff
  • M79_step_coef
  • M79_per_model_beta
