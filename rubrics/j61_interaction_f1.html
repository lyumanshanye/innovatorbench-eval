Interaction F1 — Evaluation

Scoring Scale (1-5)

1 - Worst 2 3 - Mid 4 5 - Best
1 Poor / None
2 Below Average
3 Moderate
4 Good
5 Excellent

You are an expert evaluator assessing AI agent trajectories from InnovatorBench (661 trajectories across 40 engineering tasks and 15 models). Your task is to evaluate the harmonic mean of interaction precision and checkpoint recall — whether the agent asked the right questions at the right times.

Definitions

Interaction F1: The harmonic mean of two complementary measures:

  1. Interaction Precision — Were the agent's proactive questions/requests to the user actually necessary? (Unnecessary questions waste user time and signal poor autonomy.)
  2. Checkpoint Recall — Did the agent ask at all the critical decision points where human input was needed? (Missing critical checkpoints leads to wasted effort on wrong approaches.)

Critical decision points (checkpoints): Moments where:

  • The task specification is ambiguous and multiple valid interpretations exist
  • A choice will have irreversible or expensive consequences
  • Domain expertise is needed that the agent demonstrably lacks
  • The agent is about to deviate significantly from the stated requirements
  • Resource-intensive operations depend on unverified assumptions

Necessary questions: Questions that:

  • Resolve genuine ambiguity in the task
  • Clarify requirements that materially affect the approach
  • Confirm high-stakes decisions before irreversible actions
  • Request information the agent cannot reasonably infer

Unnecessary questions: Questions that:

  • Ask about information clearly stated in the task description
  • Request trivial confirmations the agent should handle autonomously
  • Repeat questions already answered
  • Ask about implementation details that don't affect the outcome

Procedure

  1. Determine interaction mode: Check whether the trajectory contains any human interaction (user messages beyond the initial task). Classify as interactive or autonomous.
  1. For interactive trajectories:

a. Count total questions/requests the agent directed at the user (questions_asked) b. For each question, assess whether it was necessary (necessary_questions) c. Identify all critical decision points in the trajectory (whether or not the agent asked) d. Count how many critical decision points the agent missed (missed_checkpoints) e. Compute conceptual precision = necessary_questions / questions_asked f. Compute conceptual recall = checkpoints_hit / total_critical_checkpoints

  1. For autonomous trajectories (no interaction):

a. Set questions_asked = 0, necessary_questions = 0 b. Identify all critical decision points where asking WOULD have helped c. Count these as missed_checkpoints d. Assess whether the task was clear enough for full autonomy, or whether the agent made assumptions that led to wasted effort

  1. Assign score based on the overall balance of precision and recall (or missed opportunities for autonomous trajectories).

Scoring Scale (1-5)

For Interactive Trajectories

ScoreDescription
5Perfect balance: asked exactly when needed, never asked unnecessarily, never missed a critical checkpoint. Both precision and recall are near-perfect.
4Good: either slightly too many questions (1-2 unnecessary) OR missed 1 non-critical checkpoint. Overall interaction is efficient and well-targeted.
3Moderate: either asks several unnecessary questions OR misses important checkpoints. One side of the F1 is clearly weaker.
2Poor: severely unbalanced — either floods user with trivial questions (low precision) OR never asks despite critical ambiguity (low recall).
1Terrible: both precision and recall are bad — asks wrong questions AND misses critical ones. Interaction actively harms rather than helps.

For Autonomous Trajectories (No Interaction)

ScoreDescription
5No missed opportunities. The task was clear enough for full autonomy. Agent correctly identified that no human input was needed.
41 minor point where asking might have helped, but the agent made a reasonable default choice.
32-3 points where asking would have helped. Agent made assumptions that caused moderate rework or suboptimal results.
2Multiple points where asking was clearly needed. Agent proceeded on wrong assumptions causing significant wasted effort.
1Multiple critical misunderstandings that a single question would have resolved. Agent wasted most of its trajectory on incorrect interpretations.

Edge Cases

  • Task provides perfect specification: If the task description is unambiguous and complete, full autonomy is correct. Score 5 for autonomous trajectories in this case.
  • Agent asks questions but gets no response: Evaluate the quality of questions asked, not whether answers were received. Score based on whether the right questions were posed.
  • Agent asks rhetorical/self-directed questions: These do not count as interaction questions. Only count explicit requests directed at the user.
  • Implicit questions via partial output: If the agent shows partial work and asks "does this look right?", count this as a question. Assess whether it was at a meaningful checkpoint.
  • Repeated questions: If the agent asks the same question multiple times, count each instance. Repetition indicates low precision.
  • Very short trajectories: If the trajectory is very brief (< 5 actions), there may be few opportunities for checkpoints. Adjust expectations accordingly.

Output Format

Return ONLY valid JSON: {"score": N, "has_interaction": true/false, "questions_asked": N, "necessary_questions": N, "missed_checkpoints": N, "reason": "...", "evidence": [...]}

Where:

  • score: Integer 1-5 per the scale above
  • has_interaction: Whether the trajectory contains human interaction beyond the initial task
  • questions_asked: Total number of questions/requests the agent directed at the user (0 for autonomous)
  • necessary_questions: How many of those questions were actually necessary (0 for autonomous)
  • missed_checkpoints: Number of critical decision points where the agent should have asked but did not
  • reason: 1-3 sentence explanation of the score
  • evidence: Array of strings citing specific moments of good/bad interaction or missed opportunities
