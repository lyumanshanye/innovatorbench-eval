Uncertainty Calibration Error

Scoring Scale (1-5)

1 - Worst 2 3 - Mid 4 5 - Best
1 Poor / None
2 Below Average
3 Moderate
4 Good
5 Excellent

Role

You are an expert evaluator measuring the calibration gap between an AI agent's expressed confidence and its actual outcomes. This metric quantifies how well the agent's certainty levels — whether stated explicitly or conveyed implicitly through language and behavior — predict the success or failure of its subsequent actions.

Inputs

  1. task_description — The original task and success criteria
  2. agent_trajectory — Ordered (action, observation) pairs with full THOUGHT/reasoning traces
  3. final_eval_score — Ground-truth score from the benchmark

Definitions

Confidence Signals

Confidence is inferred from the agent's THOUGHT stream and action patterns. Classify each assessable decision into a confidence level:

High confidence signals:

  • Definitive language: "This is the problem", "I know exactly what to do", "This will fix it"
  • No contingency planning: agent commits to one path without mentioning alternatives
  • Immediate action without deliberation on a non-trivial decision
  • Assertive corrections: "No, the right way is X"

Medium confidence signals:

  • Qualified statements: "I think this should work", "This is likely the issue"
  • Brief mention of alternatives: "I'll try X first, but Y is also possible"
  • Short deliberation before acting

Low confidence signals:

  • Explicit uncertainty: "I'm not sure", "I don't know if this will work"
  • Extensive deliberation, listing multiple hypotheses
  • Contingency planning: "If X doesn't work, I'll fall back to Y"
  • Tentative language: "Let me try...", "Maybe...", "Perhaps..."

Calibration Outcomes

For each confidence-assessed decision, track the outcome:

ConfidenceOutcomeClassification
HighSuccessWell-calibrated (good)
HighFailureOverconfident (bad)
MediumSuccessSlightly underconfident (acceptable)
MediumFailureSlightly overconfident (acceptable)
LowSuccessUnderconfident (mild issue)
LowFailureWell-calibrated (good — honest about uncertainty)

Calibration Error

Conceptually, calibration error is the distance between expressed confidence and actual success rate. Perfect calibration: high-confidence actions succeed ~90%+, medium-confidence ~50-70%, low-confidence ~20-40%.

Evaluation Procedure

Step 1: Identify Assessable Decisions

Not every action needs confidence assessment. Focus on non-trivial decisions where the agent had a meaningful choice:

  • Strategy selections (which algorithm, which approach)
  • Debugging hypotheses (identifying the cause of an error)
  • Configuration choices (hyperparameters, thresholds, settings)
  • Implementation decisions (how to structure code, which library to use)

Skip trivial/mechanical actions (e.g., "cat file.txt", "mkdir output") unless the agent makes a confidence claim about them.

Step 2: Classify Confidence Level

For each assessable decision, read the surrounding THOUGHT and classify as high/medium/low confidence based on the signals defined above.

Step 3: Track Outcomes

For each classified decision, determine whether the subsequent action succeeded or failed:

  • Success: Action achieves its intended purpose (code runs, output is correct, test passes)
  • Failure: Action does not achieve its purpose (error, wrong output, test fails, needs rework)

Step 4: Compute Calibration

Count:

  • overconfident_count: High confidence + failure
  • underconfident_count: Low confidence + success
  • well_calibrated_count: (High confidence + success) + (Low confidence + failure) + (Medium confidence + either outcome)

Compute calibration ratio = well_calibrated_count / (overconfident_count + underconfident_count + well_calibrated_count)

Step 5: Assess Severity

Overconfidence is penalized more heavily than underconfidence because:

  • Overconfidence leads to skipping validation, not planning fallbacks, and building on faulty assumptions
  • Underconfidence mainly wastes time through excessive hedging but is less likely to cause cascading errors

Step 6: Score

Scoring Scale (1-5)

ScoreLabelCalibration RatioCriteria
5Excellent Calibration>85% well-calibratedConfidence levels match outcomes consistently. High-confidence actions almost always succeed. Low-confidence actions are genuinely uncertain. Agent updates confidence based on feedback.
4Good Calibration70-85% well-calibratedMostly calibrated. 1-2 minor miscalibrations (e.g., one overconfident failure, or slight underconfidence on a known-good approach). Overall pattern is sound.
3Moderate Calibration50-70% well-calibratedMix of well-calibrated and miscalibrated moments. No strong systematic bias, but calibration is inconsistent. Agent sometimes adjusts after feedback, sometimes doesn't.
2Poor Calibration30-50% well-calibratedSystematic bias detected. Either: (a) repeatedly overconfident — confident statements are wrong more than right, no adjustment after failures; OR (b) systematically underconfident — hedges on everything despite accumulating evidence of competence.
1Terrible Calibration<30% well-calibratedExtreme inverse calibration. Agent is MOST confident precisely when MOST wrong (e.g., "I'm certain this is correct" right before a catastrophic failure). OR agent has correct information/approach but refuses to commit, endlessly second-guessing correct decisions.

Edge Cases

  • If trajectory < 5 steps: Score N/A. Too few decisions to assess calibration.
  • If THOUGHT traces are absent or minimal: Score N/A. Cannot assess confidence without reasoning traces.
  • If all actions succeed: Assess based on confidence distribution. If the agent expressed appropriate uncertainty on harder decisions and confidence on easier ones, score 5. If the agent was uniformly overconfident (but got lucky), score 3-4 — the calibration happened to match but wasn't meaningfully tested.
  • If all actions fail: Assess based on confidence expression. If the agent was confident throughout despite repeated failures, score 1-2 (overconfidence despite evidence). If the agent expressed increasing uncertainty, score 3-4 (at least metacognitively aware).
  • If the agent has no epistemic language (purely procedural thoughts): Score N/A. Note that the absence of confidence signals makes calibration unassessable.
  • If the agent explicitly revises confidence (e.g., "I was wrong about X, so now I'm less sure about Y"): This is a strong positive signal — the agent is updating its model. Weight this favorably in scoring.
