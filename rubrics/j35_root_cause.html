J35

Root Cause Attribution

Scoring Scale (1-5)

1 - Worst 2 3 - Mid 4 5 - Best
1 Poor / None
2 Below Average
3 Moderate
4 Good
5 Excellent

Role

You are an expert evaluator determining whether you can accurately identify the root cause of failure (or suboptimal performance) in an AI agent's trajectory. This metric assesses diagnostic clarity: given the trajectory, how clearly can the fundamental problem be identified?

Inputs

  1. task_description — The original task and success criteria
  2. agent_trajectory — Ordered (action, observation) pairs with full context
  3. final_eval_score — Ground-truth score (0-100)
  4. task_stats — Best/median/mean scores from other agents on this task

Evaluation Procedure

Step 1: Identify the outcome gap

  • Compare final_eval_score to task_stats.best_eval and task_stats.median_eval
  • If score is at/near best: root cause = "no significant failure" (score N/A)
  • If score is below median or zero: proceed to root cause analysis

Step 2: Generate root cause hypothesis

Classify into one of these root cause categories:

  • PLANNING_FAILURE: Wrong strategy chosen at the outset (e.g., wrong algorithm, wrong order of operations)
  • TIME_MANAGEMENT: Ran out of budget before completing critical steps
  • LOOP_TRAP: Agent got stuck in circular/repetitive behavior
  • COMPREHENSION_FAILURE: Misunderstood the task requirements
  • RESOURCE_FAILURE: External resource unavailable (network, GPU, disk)
  • CASCADING_ERROR: Early error propagated and contaminated later steps
  • EXECUTION_BUG: Correct strategy but implementation has bugs
  • SCOPE_CREEP: Over-engineered one component at expense of the whole

Step 3: Trace the causal chain

  • Identify the FIRST step where the failure path begins
  • Trace how it propagated to the final outcome
  • Identify if there were opportunities to recover that were missed

Step 4: Assess confidence

  • How clearly does the evidence support this root cause?
  • Are there alternative explanations?

Scoring Scale (1-5)

ScoreLabelCriteria
5Clear Single Root CauseOne unambiguous root cause with clear causal chain; evidence is overwhelming
4Primary Root Cause + Contributing FactorsMain cause identified with high confidence; minor contributing factors exist
3Multiple Interacting Causes2-3 causes of similar weight; difficult to isolate a single root
2Unclear Root CauseEvidence is ambiguous; multiple conflicting hypotheses; poor observability
1UndiagnosableCannot determine root cause from available evidence

Special case: If final_eval_score is at/above task_stats.median_eval, score = N/A (no significant failure to diagnose).

Output Format

{
  "score": <1|2|3|4|5|"N/A">,
  "root_cause_category": "<PLANNING_FAILURE|TIME_MANAGEMENT|LOOP_TRAP|COMPREHENSION_FAILURE|RESOURCE_FAILURE|CASCADING_ERROR|EXECUTION_BUG|SCOPE_CREEP|NO_FAILURE>",
  "root_cause_description": "<specific description of what went wrong>",
  "first_failure_step": <int or null>,
  "causal_chain": ["<step 1 of propagation>", "<step 2>", "..."],
  "missed_recovery_opportunities": ["<opportunity 1>", "..."],
  "alternative_hypotheses": ["<alt explanation>", "..."],
  "confidence": <float 0-1>,
  "reason": "<3-4 sentence diagnosis>"
}

Examples

Example 1: Score 5 (Task 16, RL Reward, score=0)

Root cause is unambiguous: LOOP_TRAP. Agent entered a micro-editing loop on line 83-95 of qa.py (58 edits) and never progressed to training or inference. The causal chain is: edit loop → all time consumed → no training → no output → score 0.

{"score": 5, "root_cause_category": "LOOP_TRAP", "root_cause_description": "Agent entered pathological edit loop on reward function (58 edits to same lines in qa.py) without ever testing the function, consuming entire time budget", "first_failure_step": 36, "causal_chain": ["Step 36: First edit to qa.py line 83", "Steps 36-155: 58 repeated edits to same lines without compilation or testing", "Budget exhausted: no time for train→infer pipeline", "No output file produced, score = 0"], "missed_recovery_opportunities": ["After 10th edit: should have tested current version", "After 50th step: should have assessed time remaining and prioritized end-to-end run", "Think node 3 recognized need to simplify but didn't act on it"], "alternative_hypotheses": ["SCOPE_CREEP: over-engineering reward function"], "confidence": 0.95, "reason": "Crystal clear root cause. The agent's 58 edits to the same file lines constitute an unmistakable loop trap. This directly caused budget exhaustion and zero output. No alternative explanation is plausible."}

Example 2: Score 3 (Task 5, Data Cleaning, score=17.44)

Multiple interacting causes: (a) initial cleaning rules too aggressive (logic issue), (b) enhancement made it worse (regression), (c) eval non-determinism confused rollback strategy. Hard to isolate one root cause.

Enhancement attempt introduced worse heuristics rather than fixing the original issue", "first_failure_step": 5, "causal_chain": ["Step 5: Initial cleaning rules designed without analyzing what makes content 'good'", "Step 19: First eval shows 3.186/5 quality — above baseline but not high", "Steps 20-35: Enhancement adds more aggressive filters", "Step 36: Score drops to 3.122 — enhancement was counterproductive", "Step 38: Rollback attempted but third eval even lower (3.077)"], "missed_recovery_opportunities": ["Before enhancement: should have sampled and analyzed what the judge considers 'quality'", "After first score drop: should have analyzed which new filters removed good content"], "alternative_hypotheses": ["Eval non-determinism: same data scored differently across runs", "PLANNING_FAILURE: should have used LLM-based quality scoring rather than heuristic rules"], "confidence": 0.55, "reason": "Multiple interacting factors. The heuristic approach may be fundamentally limited, but the enhancement regression and eval score inconsistency (rollback scored lower than original) suggest additional confounds. Cannot isolate a single root cause with high confidence."}

Example 3: Score 4 (Task 18, Agent/Workflow, no eval)

Primary cause: TIME_MANAGEMENT. Agent spent too long debugging import/syntax errors (~45 steps) before getting the code to run. Contributing factor: code was written in one large chunk rather than incrementally tested.
