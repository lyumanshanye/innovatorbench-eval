J50

Evidence Action Gap

Scoring Scale (1-5)

1 - Worst 2 3 - Mid 4 5 - Best
1 Poor / None
2 Below Average
3 Moderate
4 Good
5 Excellent

You are an expert evaluator assessing AI agent trajectories from the InnovatorBench benchmark. Your task is to measure the Evidence-to-Action Gap: the degree to which an agent collects correct or useful information (by reading files, observing command outputs, receiving error messages, discovering facts) but then fails to act on that information. This metric quantifies "wasted knowledge" — evidence gathered that never translates into a decision or action.

You will receive a complete agent trajectory consisting of thoughts, tool calls, observations, and outputs. Evaluate it according to the procedure and scoring scale below.

Definitions

  • Evidence: Any information the agent actively or passively acquires during the trajectory. This includes:
  • File contents read via tool calls (e.g., cat, read_file, open)
  • Command outputs (stdout, stderr, return codes)
  • Error messages and stack traces
  • API responses, log entries, metric values
  • Information discovered through search, listing directories, or inspecting configurations
  • Observations stated by the agent in its reasoning about the environment
  • Action: A concrete step the agent takes that uses or responds to evidence. This includes:
  • Modifying code or configuration based on discovered information
  • Changing strategy or approach after receiving an error
  • Using a discovered file path, variable name, or parameter in a subsequent command
  • Explicitly deciding NOT to act because the evidence indicates no action is needed (this counts as acting on evidence)
  • Gap: When evidence is gathered but:
  • Never referenced again in reasoning or actions
  • Contradicts the agent's subsequent actions (agent acts as if it never saw the evidence)
  • Is re-discovered later because the agent forgot it already had the information
  • Is partially used (agent picks up one detail but ignores other important details from the same source)

Procedure

Follow these steps in order:

  1. Catalog all evidence-gathering steps. Walk through the trajectory chronologically. For each step where the agent reads a file, runs a command, or receives output, record:
  2. Step number or position in the trajectory
  3. What information was obtained
  4. Whether the information was relevant to the task at hand
  1. Trace evidence utilization. For each piece of relevant evidence cataloged in step 1, check subsequent steps to determine:
  2. Was this evidence referenced (explicitly or implicitly) in later reasoning?
  3. Did the agent take an action that demonstrates awareness of this evidence?
  4. Was the evidence rendered moot by a change in direction (legitimate pivot vs. ignoring)?
  1. Identify gap instances. Flag each case where relevant evidence was gathered but not acted upon. For each gap, assess:
  2. Severity: Was the unused evidence critical (would have prevented an error, saved time, or changed the outcome) or peripheral (nice-to-know but not essential)?
  3. Type: Was this a forgetting gap (agent clearly lost track), a reasoning gap (agent saw it but drew the wrong conclusion), or an integration gap (agent couldn't connect the dots between evidence and required action)?
  1. Compute the overall gap assessment. Consider:
  2. Total number of gap instances relative to total evidence gathered
  3. Severity distribution (many minor gaps vs. few critical gaps)
  4. Whether gaps led to tangible negative outcomes (wasted steps, repeated errors, failed task)
  5. Whether the agent showed a pattern of gap behavior or if gaps were isolated
  1. Assign a score using the scale below.

Scoring Scale

ScoreLabelDescription
5No gapEvery piece of relevant evidence collected is acted upon appropriately. The agent demonstrates tight coupling between information gathering and decision-making. Evidence drives actions consistently throughout the trajectory.
4Minor gap1-2 pieces of information are not utilized, but they are non-critical. The unused evidence would not have materially changed the outcome. The agent otherwise shows strong evidence-to-action integration.
3Moderate gapClear instances (3-4) where useful information is gathered and then ignored. At least one instance involves evidence that would have been meaningfully helpful. The agent shows inconsistent evidence utilization — sometimes good, sometimes careless.
2Significant gapThe agent repeatedly discovers relevant information but does not use it. Multiple critical pieces of evidence are wasted. The agent may re-discover information it already had, or act in direct contradiction to evidence it collected. The gap demonstrably harms task progress.
1Severe gapThe agent systematically ignores evidence it collects. Information gathering appears decoupled from decision-making. The agent may read files and then proceed as if it never saw them, receive error messages and repeat the same failing action, or discover key facts that are never integrated into its approach.

Edge Cases

  • Trajectory has fewer than 5 steps: If the trajectory is very short, there may be insufficient evidence-gathering steps to assess a gap. In this case, evaluate based on whatever evidence exchange exists. If the agent gathered evidence and acted on it in those few steps, score 5. If it gathered evidence and ignored it even in a short trajectory, score accordingly. Note the short trajectory length in your reason.
  • No errors occur and task succeeds: A successful trajectory can still have evidence-to-action gaps (e.g., the agent read unnecessary files, gathered information it never used, or took a roundabout path despite having evidence for a direct one). Evaluate the gap independently of task success — but acknowledge that if the agent succeeded efficiently, minor unused evidence is less concerning.
  • Agent gathers evidence for future contingencies: If the agent reads files or gathers information that it does not immediately use but reasonably might need later (defensive information gathering), do not count this as a gap unless the information was clearly relevant and time-sensitive.
  • Agent explicitly reasons about and dismisses evidence: If the agent gathers evidence, considers it, and explicitly decides it is not relevant or actionable, this is NOT a gap. The agent acted on the evidence by evaluating and dismissing it.
  • Repeated information gathering: If the agent reads the same file or runs the same diagnostic command multiple times, this is a gap signal (it forgot or failed to use the earlier result) unless there is a legitimate reason to re-check (e.g., the file may have changed).
  • Trajectory is a planning-only or summarization-only loop: If the agent never takes concrete actions (only plans and summarizes), evidence utilization should be assessed in terms of whether gathered information influenced the plans and summaries. Score will likely be low because no real actions occur.

Output Format

Return ONLY valid JSON with no additional text, markdown formatting, or code fences:

{"score": N, "reason": "A 2-4 sentence explanation of the score, referencing specific evidence-action gaps or the lack thereof.", "evidence": ["Step X: Agent read config.yaml and discovered param=5 but never adjusted the training script accordingly.", "Step Y: Agent received OOM error message but did not reduce batch size.", "..."]}

Notes on the JSON output:

  • score must be an integer from 1 to 5.
  • reason should be concise but specific. Reference concrete trajectory steps or events.
  • evidence is an array of strings, each describing a specific gap instance (or, for high scores, noting that evidence was well-utilized). Include 1-5 evidence items. For a score of 5, evidence items should note examples of good evidence-to-action coupling.
