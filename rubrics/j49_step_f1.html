Step-level Failure F1 — Evaluation

Scoring Scale (1-5)

1 - Worst 2 3 - Mid 4 5 - Best
1 Poor / None
2 Below Average
3 Moderate
4 Good

You are an expert evaluator assessing AI agent trajectories from InnovatorBench (661 trajectories across 40 engineering tasks and 15 models). Your task is to evaluate the overall quality of the agent's failure detection by computing the F1 score — the harmonic mean of precision (J47) and recall (J48) — which captures whether the agent accurately identifies failures without false alarms and without missing real problems.

Definitions

Step-level Failure F1 is the harmonic mean of:

  • Precision (J47): Of all steps the agent CLAIMS failed, what fraction truly failed?
  • Recall (J48): Of all steps that ACTUALLY failed, what fraction did the agent detect?

F1 = 2 (Precision Recall) / (Precision + Recall)

The harmonic mean penalizes imbalance: an agent with perfect precision but terrible recall (or vice versa) gets a low F1. Both dimensions must be adequate for a high F1.

Why F1 Matters

  • High precision + low recall = The agent is conservative. When it claims something failed, it is correct — but it misses many actual failures, building on broken foundations silently.
  • Low precision + high recall = The agent is paranoid. It catches all failures — but also falsely claims many successes are failures, wasting time "fixing" things that work and potentially breaking them.
  • High F1 = The agent has calibrated failure detection. It notices real problems and doesn't cry wolf.

Procedure

  1. Assess Precision (as in J47):
  2. Identify all failure claims by the agent.
  3. Determine which claims are true (step genuinely failed) vs false (step actually worked).
  4. Estimate precision as a proportion (0.0 to 1.0) or qualitative level.
  1. Assess Recall (as in J48):
  2. Identify all actual failures in the trajectory (from tool outputs, wrong results, downstream effects).
  3. Determine which actual failures the agent detected vs missed.
  4. Estimate recall as a proportion (0.0 to 1.0) or qualitative level.
  1. Compute F1 qualitatively:
  2. If both precision and recall are high: F1 is high (score 4-5).
  3. If one is high and the other is low: F1 is moderate-to-low (score 2-3).
  4. If both are low: F1 is very low (score 1).
  5. Remember: F1 is NOT a simple average. Precision=5, Recall=1 yields F1 around 2, not 3.
  1. Consider the practical impact: Does the agent's failure detection quality (combining both dimensions) lead to effective problem-solving? Or does imbalance (either direction) cause significant wasted effort or silent accumulation of errors?
  1. Assign a score reflecting the combined F1 quality.

Scoring Scale (1-5)

ScoreDescription
5Both precision AND recall are excellent. The agent knows exactly what failed, reports only genuine failures, and misses nothing. Failure detection is calibrated and comprehensive. This enables efficient error correction without wasted effort.
4Good balance between precision and recall. There may be a slight weakness in one dimension (e.g., 1-2 false claims OR 1-2 missed failures) but both are above adequate. The agent's failure detection reliably supports effective problem-solving.
3Moderate F1 — either precision or recall is good but the other is poor, creating a one-sided failure detection profile. Examples: the agent catches all failures but also falsely claims many successes are failures (high recall, low precision); OR the agent is accurate when it claims failure but misses half the actual failures (high precision, low recall).
2Poor F1 — both precision and recall have significant issues, OR one dimension is near-zero. The agent's failure detection provides little useful signal — it either misses failures AND makes false claims, or one dimension is so poor that the other cannot compensate (e.g., precision=0.9, recall=0.1 yields F1=0.18).
1Terrible F1 — the agent either never notices failures (recall near zero), or claims everything failed (precision near zero), or both dimensions are broken. Failure detection provides no actionable information and may actively mislead the agent's problem-solving.

Edge Cases

  • No failures and no claims: If there are no actual failures in the trajectory AND the agent makes no failure claims, both precision and recall are trivially 1.0. Score 5.
  • No failures but false claims exist: Precision is 0 (all claims are false), recall is trivially 1.0 (nothing to miss). F1 = 0. Score 1.
  • Failures exist but no claims made: Precision is trivially 1.0 (no false claims), recall is 0 (nothing detected). F1 = 0. Score 1.
  • Very few data points: If there is only 1 failure and 1 claim, and they match, that is technically perfect F1. But with so little data, cap the score at 4 unless the trajectory is genuinely short and simple.
  • Asymmetric impact: If low recall has much worse practical consequences than low precision (or vice versa) in the specific task context, note this in your reasoning but still score based on the F1 formula — this metric is deliberately balanced.

Output Format

Return ONLY valid JSON: {"score": <integer 1-5>, "precision_estimate": "<qualitative: excellent/good/moderate/poor/very_poor>", "recall_estimate": "<qualitative: excellent/good/moderate/poor/very_poor>", "reason": "<2-4 sentence justification explaining both precision and recall levels and how they combine into the F1 score>", "evidence": ["<example demonstrating precision quality (true or false failure claim)>", "<example demonstrating recall quality (detected or missed failure)>", "..."]}
