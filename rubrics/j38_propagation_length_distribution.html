Aggregates J37 (per-error propagation length, in steps) into a per-model distribution: mean, std, and p25/p50/p75/p95. Low mean + low std = errors die fast; high mean = errors live long; high std = inconsistent.

At a Glance

FieldValue
Metric IDJ38
EnglishPropagation Length Distribution
中文传播长度分布
MethodAuto / Aggregate (built on J37)
Value rangenon-negative integers (steps)
DirectionLower mean & std = healthier; high mean or std = error-handling weakness.

Formula

for each model m:
    lengths = [J37(t) for t in trajectories where model=m]
    J38_mean(m) = mean(lengths)
    J38_std(m)  = std(lengths)
    J38_p{25,50,75,95}(m) = numpy.percentile(lengths, q)

Required Inputs

Per-trajectory J37_propagation_length values keyed by model.

Procedure

  1. Run J37 on every trajectory (separate judge step).
  2. Group J37 values by model.
  3. Compute mean, std, and quartile + p95 percentiles per model.
  4. Persist as a per-model row in judge_m37_m38/m38_aggregated.json.

Worked Example

Aggregated from judge_m37_m38/results.json (per-trajectory J37) into judge_m37_m38/m38_aggregated.json. Per-model rows include {mean, std, p25, p50, p75, p95}. The current presentation page does not yet expose model-level comparison tables — open the JSON for raw values.

Edge Cases

  • Models with <5 trajectories ⇒ percentiles unreliable; flag as low-n in the output.
  • If J37 is missing for a trajectory (judge parse error), it's dropped from the aggregate.

Output Keys

  • J38_mean_per_model
  • J38_std_per_model
  • J38_p25/p50/p75/p95

Source

judge_m37_m38.py (aggregation step over J37 results). Aggregated JSON: judge_m37_m38/m38_aggregated.json.
