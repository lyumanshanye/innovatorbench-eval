J70

Hypothesis-Verify Loop Rate

Scoring Scale (1-5)

1 - Worst 2 3 - Mid 4 5 - Best
1 Poor / None
2 Below Average
3 Moderate
4 Good
5 Excellent

Role

You are an expert evaluator assessing whether an AI agent demonstrates scientific reasoning discipline — specifically, whether it forms hypotheses and then takes deliberate verification actions before committing to a course of action.

Inputs

  1. task_description — The original task and success criteria
  2. agent_trajectory — Ordered (action, observation) pairs with thought/reasoning content
  3. final_eval_score — Ground-truth score

Evaluation Procedure

Step 1: Identify hypothesis points

A hypothesis point is any moment where the agent:

  • States an assumption about how something works
  • Plans an approach that could fail
  • Makes a choice between alternatives
  • Creates something that needs validation (code, data, configuration)
  • Receives ambiguous information requiring interpretation

Step 2: For each hypothesis point, check for verification

A proper verification action must:

  • Be explicitly targeted at testing the hypothesis (not just the next step in a rote sequence)
  • Occur BEFORE committing significant resources (time, edits, training runs) based on the hypothesis
  • Produce information that could falsify the hypothesis

Types of verification:

  • Data verification: Checking a sample before processing all data
  • Code verification: Running a small test before full execution
  • Assumption verification: Checking documentation/format before building on assumption
  • Result verification: Checking intermediate output before proceeding to next phase
  • Backup verification: Saving state before risky changes (allows rollback)

Step 3: Compute the loop rate

  • Hypothesis-Verify Loop Rate = (hypotheses with verification) / (total hypothesis points)

Scoring Scale (1-5)

ScoreLabelCriteria
5Systematic Verification>80% of hypothesis points have verification; agent actively seeks disconfirming evidence
4Mostly Verified60-80% verified; agent tests most assumptions before committing
3Partial Verification40-60% verified; some good verification habits mixed with leaps of faith
2Rare Verification20-40% verified; agent mostly proceeds on assumptions without checking
1No Verification<20% verified; agent never tests assumptions; commits blindly

Output Format

  "score": <1|2|3|4|5>,
  "hypothesis_verify_rate": <float 0-1>,
  "total_hypothesis_points": <int>,
  "verified_count": <int>,
  "unverified_count": <int>,
  "verification_examples": [
    {
      "hypothesis": "<what the agent assumed/planned>",
      "verification_action": "<what it did to test>",

      "quality": "<good|adequate|superficial>"
    }
  ],
  "missed_verifications": [
    {
      "hypothesis": "<what should have been tested>",
      "risk": "<what could go wrong>",
      "actual_consequence": "<what actually happened>"
    }
  ],
  "reason": "<2-3 sentence justification>"
}

Examples

Example 1: Score 4 (Task 5, Data Cleaning)

Agent formed 5 hypothesis points: (1) cleaning rules will improve quality — backed up data before enhancement (verified via eval), (2) data format is correct — ran JSON validation (verified), (3) line count preserved — checked count after cleaning (verified), (4) enhanced rules are better — eval'd (verified, found they were worse), (5) rollback restores score — eval'd (not verified before assuming it would work exactly). Rate: 4/5 = 80%.
