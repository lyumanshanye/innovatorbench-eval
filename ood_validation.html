结论

真 m_v2 指标的组合在完全没见过的 swefactory 分布上，把 bad case 判别到 AUC 0.677（0.5=瞎猜，0.7=不错）——证明指标体系在 OOD 上仍携带成败信号，且是多指标互补 （单字段最高 0.646，组合升到 0.677）。直接回答两个课题：(a) 无先验下 true negative 可识别； (b) 指标在 OOD 上有用。

⚠️ 方法学教训：先用手搓的 11 个简化 proxy 探路，得出"AUC≈0.55、指标没用"的假负结论； 换真 adapter 全量跑后组合 AUC 0.677、单指标也普遍更高（M74 0.553→0.608，M15 0.531→0.610）。 proxy 系统性低估——判定指标价值必须用真 adapter，不能用手搓近似。

反直觉深挖：推理覆盖是"挣扎症状"而非"能力信号"

步级分析发现失败组的 M71 推理覆盖反而更高。在 95,828 条上深挖，这个悖论稳固且可解释：
0.452
高推理覆盖组 pass 率
0.696
低推理覆盖组 pass 率
−0.14
控探索后 推理-成败偏相关

推理越多，成功率越低（0.452 vs 0.696），且控制探索量后仍负相关(−0.14)。 推理×探索的 2×2 分层显示，最大的杠杆是探索聚焦度而非推理量：

组合pass 率n
想得少 + 聚焦探索（最好）0.70735,185
想得少 + 过度探索0.65212,729
想得多 + 聚焦探索0.56411,266
想得多 + 过度探索（最差）0.42636,648

机理：M71 推理覆盖是"挣扎症状"不是"能力信号"——难任务触发更多推理，高推理是"卡住了在硬想"的表现， 不是解题技巧。在每个探索层级里高推理都underperform，排除了"推理只要聚焦就有用"。 直接的训练含义：绝不能把推理覆盖当 reward 目标（会激励模型模仿挣扎/空想）——这条直接约束了 PROC-IF 的奖励设计。

诚实的边界

  • 组合 AUC 0.677 是有用但非强分类器；单指标最高 0.646，靠组合互补拉高。
  • M19 系列、M54/M16 等别名字段测同一构造（表中已去重展示）。
  • 这是 pass/fail 预测；过程指标本就不是成败预言机（过程⊥结果），能到 0.677 已超预期。
  • 数据源 swefactory v1(46,434)+v2(49,394)，OmniSWE 管线产出，SWE-bench 真验证器判 passed。
